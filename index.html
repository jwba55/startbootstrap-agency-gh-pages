개발자 리소스 공유 커뮤니티(코딩기딩)

개발자가 자신이 만든 리소스를 공유 및 판매 가능하도록 한 플랫폼입니다.

  • 프로젝트 주제 선정이유:
    팀원 중 웹 개발을 처음 접하는 분이 있어, 개발자 간의 리소스를 공유 및 판매하는 커뮤니티를 만들어보면 팀원들이 개발자로서 성장하는데 도움이 될 것 같아 주제를 선정하게 되었습니다.
  • 주요 트러블 슈팅:
    1. 자체 인증 시스템 구현의 어려움
    1. 자체 암호화 로직 구현 중 로그인
      • 사용자가 입력한 비밀번호와 함께 특수문자와 그 좌표를 DB에 저장하여 불러옴으로 해결
    2. 인프라 구축 및 배포 자동화
    1. EC2 인스턴스에 OracleDB 설치 및 설정 문제
      • Oracle에서 제공하는 Docker 이미지를 불러옴으로 해결
      • Docker Compose를 활용하여 DB가 실행된 후 애플리케이션이 실행되도록 설정
    2. Tomcat 포트 변경이 적용되지 않는 문제
      • GitHub Actions에 Server.xml을 만들어두고 Dockerfile에서 파일을 복사하도록 구성하여 해결
    3. NginX SSL 설정 문제
      • Let's Encrypt의 Certbot을 사용하여 SSL 인증서를 자동으로 발급·갱신하도록 설정하여 해결
  • 기술 스택:
    (아이콘을 클릭하면 선택 이유가 나옵니다.)
    Spring Legacy
    OracleDB
    Maven
    MyBatis
    JSP
    JSTL
    HTML
    CSS
    JavaScript
    AWS
    NginX
    Docker
    GitHub Actions
  • 기간:
    2025.04 ~ 2025.06

  • 팀원:
    정지욱, 김택수, 장민규
    링크:
    1. 상세 코드 보기
    2. 사이트 이동
  • 주요 기능:
    • 회원가입/로그인, 상품 조회, 리소스 업로드/다운로드, 결제 시스템, 댓글 및 평점 시스템, 출석체크, 랭킹 시스템, 게시판
    담당 역할:
    1. 1. 기획
      • 요구사항 분석 및 기능 명세서 작성
      • DB 설계 및 ERD 작성
    2. 2. 백엔드 개발 전반 (구현, 테스트, 배포)
      • 자체 인증 및 인가 시스템 구현 (상세보기)
      • 회원관리, 리소스 업로드, 상품 조회, 판매 내역, 랭킹 시스템
      • 인프라 구축 및 배포 자동화
      • 팀원 코드 리뷰 및 기술 지원
    3. 3. 프론트엔드(기능 별 분배)
      • JSP 기반 UI 개발
      • 사용자 경험(UX) 개선

Users userId roleId 사용자 계정의 핵심 정보(가입일, 닉네임 등)를 관리합니다.
UsersRole roleId — 사용자에게 부여되는 역할(Role) 종류를 정의합니다.

NewRegex newRegexId userId 새로운 특수문자와 해당 특수문자의 좌표를 기록합니다.
ResourceCategory resourceCategoryId — 리소스 상품의 상위 카테고리를 분류합니다.
ResourceSubCategory resourceSubCategoryId resourceCategoryId 상위 카테고리에 속하는 하위 카테고리를 정의합니다.
ResourceShop itemId userId, resourceSubCategoryId 상점에서 판매하는 리소스 상품의 상세 정보(가격, 내용, 등록자)를 관리합니다.
RESOURCEFILE RESOURCEFILEID itemId 리소스 상품에 연결된 파일 정보를 저장합니다.
Ranking rankId itemId 상품별 월간/주간/일간/전체 순위 정보를 기록합니다.
Counting countingId itemId 상품별 판매량의 카운트 통계를 관리합니다.

Orders ordersId userId 사용자별 주문의 기본 정보(주문일자, 총 금액) — 주문 헤더 테이블입니다.
OrdersDetails ordersDetailsId ordersId, itemId 하나의 주문에 포함된 개별 상품의 상세 내역(수량, 당시 가격)을 기록합니다.
BuyPoint pointId userId 사용자의 포인트 구매 및 획득 내역을 기록합니다.
ForumBoard boardId — 운영되는 게시판의 카테고리를 관리합니다.
ForumPost postId boardId, userId 게시글 정보(제목, 내용, 작성자)를 관리합니다.
ForumPostFile filePostId postId 게시글에 첨부된 파일 정보를 관리합니다.
ForumPostReview commentId postId, userId 게시글에 대한 댓글/리뷰 정보를 기록합니다.
UserAttendanceDetail attendId userId 사용자 별로 출석한 날짜를 상세히 기록합니다.
AttendanceBonus 복합키 userId 사용자에게 월별 출석 보너스가 지급되었는지 여부를 관리합니다.

  • 메인페이지:

    메인페이지에서는 최신 리소스와 게시글 등을 한 번에 볼 수 있도록 했습니다.

  • 회원가입/로그인 페이지:

    자체 인증 및 인가 시스템을 구현하여 사용자가 안전하게 회원가입하고 로그인할 수 있도록 하였습니다.
    비밀번호 암호화 로직을 직접 구현하여 보안성을 높였습니다.

  • 출석체크 페이지:

    사용자가 출석체크를 통해 포인트를 획득할 수 있도록 하였습니다.
    출석 현황을 달력 형태로 보여주어 사용자가 자신의 출석 기록을 쉽게 확인할 수 있도록 하였습니다.

  • 상품 페이지:

    사용자가 각 카테고리 별로 분류해서 상품을 볼 수 있도록 1차, 2차 카테고리를 선택해서
    볼 수 있도록 페이지 랜딩 시 전체 목록을 불러온 다음 JavaScript를 통해
    선택된 카테고리에 해당하는 상품만 볼 수 있도록 하였습니다.

  • 상품 업로드 페이지:

    사용자가 자신이 만든 리소스를 업로드하고 판매할 수 있도록 하였습니다.
    다중 파일 첨부 기능을 통해 리소스 파일을 업로드할 수 있도록 하였습니다.

  • 상품 상세 페이지:

    상품의 상세 정보 및 등록된 파일을 확인하고, 리뷰를 남기거나 구매할 수 있도록 하였습니다.

  • 게시판 페이지:

    게시판 형태에 따라 자유게시판, 공지게시판, 취업정보 게시판으로 분류해두고
    각 게시판 성격에 맞게 첨부파일을 올리거나 댓글을 남길 수 있도록 하였습니다.

  • 구매/판매내역 페이지:

    사용자가 자신의 구매 및 판매 내역을 한 눈에 볼 수 있도록 하였습니다.
    각 내역 별로 상세 정보를 확인할 수 있도록 하였습니다.

SNS 프로젝트(완두콩)

사이트 명인 ‘완두콩’ 또한 프로젝트의 부제목인 ‘추억을 심다‘에 맞게
그때 그때의 싱그러운 추억을 심고 가꾸는 것을 성장하는 식물에 빗대어 표현하였습니다.

  • 프로젝트 주제 선정 이유:
    추억이 담긴 싸이월드의 몰락이 아쉬웠던 마음이 컸고 SNS에는 다양한 기능들이 있어서
    이 프로젝트를 진행하며 개발자로서의 실력 향상에 매우 도움될 것이라 생각했습니다.
  • 프로젝트 설명:
    이 프로젝트의 주된 목표는 자극성을 줄이고 추억을 기억하고 회상하는 것으로
    사용자들이 자유롭게 자신의 개성을 표현하고 타인과 소통할 수 있는 시스템을
    제공합니다.
